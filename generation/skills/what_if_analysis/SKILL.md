## Analytical Framework: What-If Analysis

Apply What-If Analysis to generate hypotheses from the source context.
This method uses the reframing power of assumed outcomes to break through
analytical anchoring.

### Core Concept

When analysts believe an outcome is unlikely, they tend to dismiss it without
serious examination. What-If Analysis reverses this by ASSUMING the outcome
has already occurred, then asking "how could this have happened?" This
reframing transforms the question from probability estimation (which is
vulnerable to anchoring) to causal reconstruction (which engages deeper
analytical thinking).

### Procedure

1. **Identify the anchor**: Read the context and identify the dominant
   explanation — the one that feels most natural or obvious. This is the
   explanation you need to look BEYOND.

2. **Formulate 2-3 "What If" scenarios**: For each scenario, assume a
   DIFFERENT explanation is true. Frame it concretely:
   - "What if the outcome was not caused by [obvious cause] but by
     [alternative cause]? It's now confirmed — how did it happen?"
   - Choose alternatives that are genuinely different from the anchor,
     spanning different categories of explanation (e.g., if the anchor is
     a technical cause, consider human factors, organizational dynamics,
     or design-level issues).

3. **Backward reasoning for each scenario**: For each "What If," work
   backward from the assumed outcome to the present evidence:
   - **Reinterpret evidence**: Which facts in the context are ALSO
     consistent with this alternative scenario? Which facts would need
     to be reinterpreted?
   - **Identify missing evidence**: What evidence SHOULD exist if this
     scenario is true but is not mentioned in the context? Is its absence
     because it wasn't looked for, or because it doesn't exist?
   - **Construct the causal path**: Build a step-by-step chain from
     initial conditions to the observed outcome through this alternative
     mechanism.
   - **Identify a trigger**: What specific event or condition would
     have set this alternative chain in motion?

4. **Generate indicators**: For each scenario, list 2-3 observable indicators
   that would help distinguish between the anchor explanation and this
   alternative. What evidence, if found, would confirm or refute each
   scenario?

5. **Assess plausibility**: After the backward reasoning exercise, honestly
   assess:
   - Which scenarios survived the reasoning process as genuinely plausible?
   - Which required too many unsupported assumptions?
   - Did any scenario reveal evidence that the anchor explanation cannot
     adequately explain?

6. **Select the strongest**: Present the scenario(s) that emerged as most
   plausible from the backward reasoning, alongside the anchor if it
   remains strong.

### Quality Criteria

- At least 2 genuinely distinct "What If" scenarios must be explored,
  not variations of the same idea.
- Backward reasoning must engage specifically with the evidence in the
  context, not reason in the abstract.
- Each scenario must include specific indicators that would distinguish
  it from alternatives.
- The plausibility assessment must be honest — scenarios that don't
  survive backward reasoning should be acknowledged as eliminated,
  not artificially kept alive.
